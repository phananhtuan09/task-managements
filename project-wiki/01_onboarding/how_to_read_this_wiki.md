# How to Read This Wiki

Use this reading order for both onboarding and change analysis.

1. Read [README](../README.md).
2. Read [INDEX](../INDEX.md).
3. Read [GOVERNANCE](../GOVERNANCE.md).
4. Read [project_overview](project_overview.md).
5. Read [business_rules](../02_requirements/confirmed/business_rules.md).
6. Read feature docs for the target domain. Until canonical feature docs exist, start at [03_features/README.md](../03_features/README.md).
7. Read flow docs for the affected area. Until canonical flow docs exist, start at [04_flows/README.md](../04_flows/README.md).
8. Read relevant ADRs before major changes. Until ADRs exist, start at [06_decisions/README.md](../06_decisions/README.md).

## Targeted Use

- For a broad orientation, stop after step 5.
- For a feature change, continue through steps 6 to 8 for the relevant area.
- If a linked area is still planned, return to [INDEX](../INDEX.md) and use the nearest folder entry point.
