# How to Read This Wiki

Use this reading order for product onboarding and feature clarification.

1. Read [README](../README.md).
2. Read [INDEX](../INDEX.md).
3. Read [project_overview](project_overview.md).
4. Read [business_context](business_context.md).
5. Read [business_rules](../02_requirements/confirmed/business_rules.md).
6. Read [product_scope](../02_requirements/confirmed/product_scope.md).
7. Read feature docs for the target domain. Start at [03_features/README.md](../03_features/README.md).

## Targeted Use

- For a broad orientation, stop after step 5.
- For a feature change, continue to the relevant feature spec.
- If no feature doc exists yet, return to [INDEX](../INDEX.md) and add or update the nearest product-facing doc first.
