# Requirement Drafts

Use this folder for partial or still-evolving requirement notes that are not ready to act as durable truth.

## When to Use This Area

- early notes that still need shaping
- requirement changes that need discussion before promotion
- temporary working drafts that should not be treated as canonical

## Promotion Path

1. Start with a draft or change request when the topic is incomplete.
2. Move the content into the appropriate durable doc once the topic and scope are clear.
3. Keep status handling and confirmation behavior aligned with [GOVERNANCE.md](../../GOVERNANCE.md).

## Rules of Thumb

- Draft content is non-canonical by default.
- If a topic becomes durable, move it into the right requirement, feature, flow, or system surface.
- If a draft is abandoned, keep any needed trace explicit instead of letting it look current by accident.
